# OpenAPI und Swagger in NestJS

Eine funktionsfähige REST-API in NestJS zu bauen, ist nur die halbe Arbeit. Im Moment verarbeiten deine Endpunkte die Daten zwar korrekt, aber für andere bleiben sie eine Blackbox. Ein Frontend-Entwickler, der deine API nutzen möchte, muss deine DTOs rückwärts analysieren, um herauszufinden, welche JSON-Payloads erforderlich sind, oder sich per Versuch und Irrtum an deine Fehlerantworten herantasten. (Und seien wir ehrlich: Meistens schreibt er dir dann einfach auf Slack und unterbricht deinen Flow.)

Der Standardansatz für dieses Problem ist ein separates, von Hand geschriebenes Dokument, das jeden Endpunkt beschreibt. Aber seien wir ehrlich: So etwas hält niemand aktuell. Sobald du ein Update an einem Controller mergst, ist diese manuelle Dokumentation veraltet, und du bist wieder dabei, Payloads im Chat zu erklären.

OpenAPI ist die moderne Alternative. Es handelt sich um eine Spezifikation zur Beschreibung einer REST-API in einer strukturierten, maschinenlesbaren Datei (YAML oder JSON). Weil diese Datei maschinenlesbar ist, können Tools sehr nützliche Dinge damit tun: interaktive Dokumentation darstellen, typisierte Client-SDKs generieren, Contract-Tests in der CI ausführen oder Mock-Server starten. Die Swagger-Toolchain ist die am weitesten verbreitete Sammlung solcher Werkzeuge.

Die interessante Frage ist, wie sich die OpenAPI-Datei synchron zum Code halten lässt. Wenn man sie von Hand schreibt, hat man dasselbe Veraltungsproblem wie bei einem Markdown-Dokument. Die NestJS-Antwort lautet natürlich: Decorators! Sie ermöglichen es, das OpenAPI-Dokument direkt aus dem Code abzuleiten. Das Paket `@nestjs/swagger` liest dieselben Decorators, die du ohnehin schon in Controllern und DTOs verwendest (`@Get`, `@Post`, `@IsString`, Typannotationen), und setzt daraus beim Start der Anwendung das OpenAPI-Dokument zusammen. Einige zusätzliche `@Api*`-Decorators ergänzen die Informationen, die TypeScript nicht selbst ableiten kann, zum Beispiel Response-Codes und menschenlesbare Beschreibungen.

In dieser Session schauen wir uns den OpenAPI-Standard und sein zugrunde liegendes Dokumentformat genauer an, bevor wir ihn direkt in ein NestJS-Projekt integrieren. Am Ende setzt du das Gelernte in der Challenge praktisch um, indem du für die Cyber-Chat-API aus den vorherigen Modulen eine live laufende, interaktive Swagger-Oberfläche erzeugst.
